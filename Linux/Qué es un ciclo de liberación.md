El desarrollo del núcleo es un proceso continuo, pero en una cierta etapa de desarrollo, se lanza una nueva versión del kernel cuando se completan una serie de funciones y correcciones de errores. Estas nuevas versiones se llaman "lanzamientos de kernel". Linus Torvalds lanza un nuevo kernel, luego abre una ventana de fusión de dos semanas. Durante este período de ventana de fusión, extrae el código para la siguiente liberación del mantenedor del subsistema. El mantenedor del subsistema envía una solicitud de extracción git firmada por Linus durante o antes de la ventana de fusión. Todas las principales características nuevas se agregan al kernel durante el período de ventana de fusión. Más de 10,000 cambios (parches) serán arrastrados al árbol de Linus en las últimas dos semanas, y el primer candidato a la liberación (**rc1)** será liberado después del final.

El ciclo de liberación se destina al modo de solo corrección de errores, y una serie de candidatos de liberación (**rc**) se liberan de Linus. Este ciclo se repite hasta que **rc2** se libera una semana después de **que se** libera , **y rc3** se libera una semana después, y todas las correcciones y regresiones de errores importantes (si las hay) se resuelven.

El nuevo ciclo comienza una semana antes del lanzamiento y en el _período de tres_ semanas _de silencm muyio_ que dura hasta la ventana de fusión de dos semanas. Los mantenedores y los principales contribuyentes están ocupados preparando árboles para enviar solicitudes de extracción a Linus. Tenga en cuenta que el período de silencio no es formal y puede ser manejado de manera diferente para cada subsistema. Este período no se anuncia específicamente, por lo que los nuevos desarrolladores pueden sentir que la comunidad es lenta para responder.
